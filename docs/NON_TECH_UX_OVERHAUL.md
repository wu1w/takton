# Takton 非专业用户体验优化设计 v1.0

> 目标：让从没用过 AI Agent 的普通用户也能流畅使用 Takton
> 覆盖：Settings 简化 / 侧边栏名词提示 / 工作流自然语言创建 + 模板 + Cron+会话联动

---

## 一、总体策略：三层漏斗

```
第一层：零配置即用
  ┌─────────────────────────────────────────────┐
  │  打开网站 → 直接聊天 → 得到回答              │
  │  └ 无需注册、无需配模型、无需理解任何概念     │
  └─────────────────────────────────────────────┘
               │ 用久了想深入...
               ▼
第二层：按需配置
  ┌─────────────────────────────────────────────┐
  │  设置页精简为 3 项核心选择                    │
  │  侧边栏名词旁有 ? 解释                       │
  │  工作流用"说"的就能创建                      │
  └─────────────────────────────────────────────┘
               │ 需要更强大...
               ▼
第三层：高级功能
  ┌─────────────────────────────────────────────┐
  │  全部参数模式（现有 Settings）               │
  │  DAG 画布拖拽编辑                           │
  │  MCP / 自定义工具                           │
  └─────────────────────────────────────────────┘
```

## 二、优化项一：Settings 分层设计

### 2.1 现状

Settings 页 654 行，88 次出现 provider/model/API key。用户需要理解：
- 什么是"服务商"（provider）
- 什么是"模型"（model）
- 什么是 API Key
- 什么是 base_url
- 怎么连接测试

### 2.2 改造方案

**双层结构**：普通模式（默认） + 高级模式（右上角切换）

**普通模式**：

```
┌── 设置 ─────────────────────────────────────────────┐
│                                                      │
│  ┌── AI 能力 ───────────────────────────────────┐   │
│  │                                                 │   │
│  │  选择接入方式                                    │   │
│  │                                                 │   │
│  │  ┌── 一键登录（推荐） ────────────────────┐    │   │
│  │  │  🤖 OpenAI     🤖 Anthropic            │    │   │
│  │  │  🤖 Google     🤖 阿里云百炼           │    │   │
│  │  │  点击后用浏览器授权，自动配置完成         │    │   │
│  │  └────────────────────────────────────────┘    │   │
│  │                                                 │   │
│  │  ── 或者手动配置 ──                             │   │
│  │                                                 │   │
│  │  服务商: [OpenAI ▼]                             │   │
│  │  密钥:   [sk-... •••••••• ]  [👁]               │   │
│  │  模型:   [gpt-4o ▼]  [🔄 拉取列表]              │   │
│  │                                                 │   │
│  │  ▸ 高级选项  ← 默认折叠                          │   │
│  │    服务地址  [https://api.openai.com/v1     ]   │   │
│  │    创意度    [0.7 ───●──────────]               │   │
│  │    最大回复  [4096]                              │   │
│  └─────────────────────────────────────────────────┘   │
│                                                      │
│  ┌── 个人偏好 ─────────────────────────────────┐    │
│  │  回复风格: [均衡 ▼]                          │    │
│  │  语言:     [中文 ▼]                          │    │
│  │  头像昵称: [修改]                            │    │
│  └─────────────────────────────────────────────┘    │
│                                                      │
│  ▸ 高级设置  ← 默认折叠                              │
│    这里是现有完整参数列表（llm/embedding/reranker/...）│
│                                                      │
└──────────────────────────────────────────────────────┘
```

**关键设计**：

1. OAuth 一键登录按钮放最显眼位置——点了就配好，用户零学习
2. 手动模式精简为 `服务商 + 密钥 + 模型` 三行——对标手机 App 的简洁度
3. "高级选项"默认折叠——放 temperature / base_url 等不常用参数
4. "高级设置"第二层折叠——放 embedding / reranker / qdrant 完全技术参数
5. 现有"全部参数"模式改名"高级模式"保留，右上角切换

### 2.3 后端改动

- OAuth 回调路由（各供应商 OAuth 流程）
- 设置按 scope 分组：`user_preferences` / `llm_basic` / `llm_advanced` / `technical`

## 三、优化项二：侧边栏名词提示

### 3.1 现状

侧边栏 12 项导航，其中大量技术术语无解释：
- 心智配置、画像、上下文、MCP、定时任务、工具

### 3.2 改造方案

每个名词旁加一个小问号 `?` 图标，hover 显示解释气泡。

```
├ Agent
│  ├ 心智配置 [?]
│  ├ 工具     [?]
│  ├ MCP      [?]
│  ├ 画像     [?]
│  ├ 上下文   [?]
│  └ 定时任务 [?]
├ 记忆
│  ├ 知识库   [?]
│  └ Wiki图谱 [?]
```

**提示词表**：

| 名词 | 提示内容 |
|------|---------|
| 心智配置 | 调整 AI 的性格和思维方式，让它更符合你的使用习惯 |
| 工具 | 给 AI 配置它能调用的能力，比如搜索网页、读写文件、运行代码 |
| MCP | 通过标准协议连接外部服务和数据源，扩展 AI 的能力边界 |
| 画像 | 管理 AI 在不同任务中的角色设定，比如"翻译助手""代码审查员" |
| 上下文 | 查看和管理 AI 当前记忆的信息，控制它能看到什么 |
| 定时任务 | 让 AI 在指定时间自动执行任务，比如每天早上生成日报 |
| 知识库 | 上传文档，AI 会自动阅读并记住内容，回答时可以参考 |
| Wiki 图谱 | 以图谱形式管理和查看 AI 的知识关联网络 |

**实现**：一个 `<Tooltip>` 组件包裹导航标签，后端提供词条数据源（硬编码即可，变更极少）。

> AI 自动生成工作流，用户确认即可使用。

### 4.4 WorkflowChatSession

这是一个轻量化的 Agent Loop，专门用于工作流对话创建。

```python
class WorkflowChatSession:
    """
    工作流对话创建 Session

    和主聊天不同的地方：
    - 工具集固定为工作流操作工具
    - state 中包含当前工作流的 DAG（对话过程中持续变化）
    - 不加载 RAG / 上下文无关知识
    - 可随时"保存"将 DAG 写入 Workflow 表
    """

    def __init__(self, user_id, session_id=None):
        self.user_id = user_id
        self.session_id = session_id or str(uuid.uuid4())
        self.messages: list[dict] = []       # 对话历史
        self.current_dag: dict = EMPTY_DAG   # 正在构建的 DAG
        self.workflow_meta: dict = {}         # 名称/描述等
```

### 4.5 对话到工作流的映射规范

为了让 LLM 正确生成 DAG，需要提供节点类型定义作为上下文：

```python
NODE_TYPE_DEFINITIONS = """
可用节点类型：

1. input — 工作流输入
   - 子类型: manual（手动触发）/ cron（定时触发）/ webhook（外部事件）
   - cron 配置: { "trigger_type": "cron", "schedule": "0 9 * * *" }

2. llm — 调用大语言模型
   - 配置: { "prompt": "...", "model": "默认", "temperature": 0.7 }

3. rag — 知识库检索
   - 配置: { "query": "...", "top_k": 5, "collection": "knowledge_base" }

4. http — HTTP 请求
   - 配置: { "url": "...", "method": "GET|POST", "headers": {}, "body": {} }

5. python — 执行 Python 代码
   - 配置: { "code": "..." }

6. condition — 条件分支
   - 配置: { "expression": "input.score > 0.5" }

7. output — 工作流输出
   - 子类型: return（返回结果）/ message（发送消息）/ webhook（回调）
"""
```

## 七、关键设计决策

1. **工作流聊天和主聊天共享模型**——用户不需要为工作流单独配模型
2. **对话记录持久化**——用户隔天回来可以继续对话完善工作流，不需要重新描述
3. **流式构建展示**——AI 每生成一个节点/边，前端实时展示，用户有"看着 AI 帮我搭"的体感
4. **保存后仍然可对话修改**——保存不是终点，用户可以说"再加一个步骤"继续迭代
5. **对话创建和画布编辑并存**——两种方式不互斥，用户随时可以切换到画布微调
6. **模板作为对话创建的快捷入口**——用户点模板后，AI 自动将模板 DAG 加载到对话中，可以说"在这个基础上加上邮件通知"
