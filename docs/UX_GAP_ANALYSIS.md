# Takton 零基础用户体验差距分析

> 对标用户群：办公室文员、行政、财务、金融从业者
> 对标产品：豆包、元宝、千问（用户已建立使用习惯）
> 核心原则：用户不需要理解技术概念就能完成所有日常工作

---

## 一、豆包/元宝/千问 的用户心智模型

### 1.1 用户对 AI 的认知

```
普通用户理解的 AI：
  ┌─ 打开 App → 打字/说话 → 得到回答 → 完事 ─┐
  └────────────────────────────────────────────┘

普通用户不理解的：
  ✗ 模型（Model）—— "AI 还有不同型号？"
  ✗ 供应商（Provider）—— "豆包不就是豆包吗，哪来供应商"
  ✗ API Key—— "这是密码吗？为什么需要这个？"
  ✗ 会话（Session）—— "聊天记录就聊天记录，什么叫 session"
  ✗ 上下文窗口—— "什么叫 128K？"
  ✗ Token—— "一个字还分好几种？"
  ✗ 向量/Embedding—— "完全不知道在说什么"
  ✗ 工具/Tool—— "AI 不就该会这些吗？"
```

### 1.2 用户期望的产品形态

| 维度 | 豆包/元宝/千问 | Takton 当前 |
|------|---------------|------------|
| 首次打开 | 即开即用，零配置 | 注册/登录 |
| 首页 | 输入框 + 几个推荐场景 | 空白会话 + 侧边栏 |
| 输入方式 | 打字、语音、图片、文件 | 打字为主 |
| 功能入口 | 底部/侧边 3-5 个场景标签 | 侧边栏 12 个导航项 |
| 设置 | 设置=改个名字换个头像 | 设置=配置供应商/模型/API Key |
| 上传文件 | 拖拽即可 | 拖拽上传+手动索引 |
| 搜索 | 一个开关"联网搜索" | 需配置搜索引擎 |
| 帮助 | 默认就能用，不需要帮助 | 不用帮助根本用不了 |

## 二、Takton 当前体验的「七宗罪」

### 罪一：注册/登录墙

**现状**：访问网站 → 跳转登录页 → 输入邮箱/用户名/密码 → 注册/登录 → 才看到主界面

**用户视角**："我就要问个问题，为什么要注册？豆包打开就能用。"

**问题本质**：多用户系统是 Takton 的技术优势，但不应该成为用户的使用门槛。

**改造方向**：
- **访客模式**：首次打开直接进入对话页，零门槛开聊
- 用户身份在需要保存数据时才提示"创建一个名字以便保存你的聊天记录"
- 参考 Notion/飞书：先给 workspace，注册是后面的事

### 罪二：侧边栏 12 个技术术语

**现状**：

```
工作区         Agent          记忆          系统
├ 任务         ├ 心智配置       ├ 知识库       └ 设置
├ 设备         ├ 工具          └ Wiki图谱
└ 工作流       ├ MCP
              ├ 画像
              ├ 上下文
              └ 定时任务
```

**用户视角**：
- "心智配置"—— 什么？AI 还有心智？
- "画像"—— 这是在画画吗？
- "上下文"—— 什么上下文？
- "MCP"—— 三个字母，直接劝退
- "定时任务"—— 这个我大概懂...但为什么要定时？

**改造方向**：
- **合并到 4-5 个非技术标签**：
  ```
  💬 对话（默认首页）
  📁 文档（知识库 + 上传文件）
  🤖 技能（Skill 模板市场，替代心智配置 + 工具 + MCP + 画像）
  ⏰ 自动化（定时任务 + 工作流，合并为一个入口）
  ⚙️ 设置（所有配置，但默认只有必要的两项）
  ```
- 把"知识库"和"Wiki 图谱"合并为"文档"
- 把"工具/MCP/画像/心智配置"合并为"技能"——对外统一叫"添加 AI 能力"
- 所有技术性标签放进"设置 > 高级"

### 罪三：Settings 暴露技术细节

**现状**：Settings 的核心操作是"选服务商 → 填 API Key → 拉模型列表 → 保存"

**用户视角**：这些全部看不懂。

**改造方向**（和之前的模型池设计一致）：

![Settings 页面的模型选择](settings-mockup.png)

但对于非技术用户，Settings 不应该展示这些。

**两步走**：
1. **预配置模式（管理员角色）**：系统管理员/部署者配置好所有模型，普通用户只看到可选列表
2. **普通用户模式下**：Settings 只有 3 项：
   ```
   ⚙️ 设置
   ├ AI 能力：选择你要用的智脑 [推荐 ▼]
   │          AI 回复风格 [平衡 ▼]
   ├ 个人信息：头像、昵称
   └ 关于：版本、帮助
   ```
   "推荐"背后自动路由到管理员预置的最佳模型。用户不需要知道什么是 provider/model。

### 罪四：知识库需要手动索引

**现状**：上传文档 → 手动点"向量索引" → 等待处理 → 看到"已索引 12 个分块"

**用户视角**："我上传了不就行了吗？为什么还要点索引？分块是什么？"

**改造方向**：
- **上传即用**：文件拖拽上传后自动后台索引，不打断用户
- 索引状态用进度条展示，但不要求用户等待
- 把"向量""分块""Embedding"这些词从 UI 中全部抹掉
- 用户只需要知道"📄 我的文档 → ✅ AI 已阅读"

### 罪五：工作流概念太重

**现状**：工作流需要拖拽节点（input/llm/rag/python/http/条件/循环）

**用户视角**："我就要每天 9 点让 AI 帮我整理邮件。为什么要画流程图？"

**改造方向**：
- **场景模板优先**：用户说"每日早报"→ 系统自动生成对应工作流
- 模板市场：预设 10-20 个常用场景（每日早报、周报生成、数据汇总、客户回复...）
- 画布编辑器作为"高级模式"隐藏在后面
- 参考 IFTTT 的模式："如果 [时间/事件] 就 [执行动作]"

### 罪六：聊天页缺少消费级特性

**对比豆包/元宝/千问**：

| 功能 | 豆包 | Takton |
|------|------|--------|
| 语音输入 | ✅ 点击麦克风 | ❌ |
| 语音播放回复 | ✅ 自动朗读 | ❌ |
| 场景预设（写朋友圈/翻译/润色） | ✅ 主页就有 | ❌ |
| 推荐问题 | ✅ 每次打开都有 | ❌ 空白 |
| 图片理解 | ✅ 上传即理解 | ⚠️ 飞书通道有 |
| 分享对话 | ✅ 生成分享图 | ❌ |
| 对话标题 | ✅ 自动命名 | ✅ 有 |
| 连续对话 | ✅ 默认 | ⚠️ session 概念 |
| 手机适配 | ✅ | ❌ |

### 罪七：没有"场景感"

豆包打开后看到的不是空白输入框，而是：

```
你今天想做什么？
[写一篇朋友圈] [翻译这段文字] [润色一下这段话] [总结这篇文章]
```

Takton 打开后是一个空白会话。用户不知道"能做什么""该从哪里开始"。

## 三、改造路线图

### 总体原则

> **一层皮、两层肉**
>
> 表面层（皮）：完全消费级，对标豆包/元宝/千问的操作习惯
> 中间层（肉）：用 Takton 的技术栈实现，但用户无感知
> 底层（骨）：当前的 Agent Loop / 工作流 / RAG / 多供应商 不动

### Phase 1 — 表面层改造（能让用户直接感知的变化）

#### 1. 首页 → 场景化对话页

```
┌────────────────────────────────────────────┐
│  Takton ······ 🔔 ······ 👤 张三          │
├────────────────────────────────────────────┤
│                                            │
│            🤖 你好！今天想做什么？           │
│                                            │
│    ┌── 推荐场景 ──────────────────┐        │
│    │  ✍️ 写周报  📊 分析数据      │        │
│    │  📝 润色文字 🔍 总结文档     │        │
│    │  🌐 搜索信息 🎨 做PPT       │        │
│    └──────────────────────────────┘        │
│                                            │
│    📎 上传文件或拖拽到此处                  │
│    ┌──────────────────────────────────┐    │
│    │                                  │    │
│    │  [🎤 语音]  [发送消息...]  [🌐]  │    │
│    └──────────────────────────────────┘    │
│                                            │
│    当前 AI: 智能推荐 · ℹ️ 切换             │
└────────────────────────────────────────────┘
```

关键改动：
- 聊天即首页，没有侧边栏打扰（把导航折叠成左上角汉堡菜单）
- 推荐场景按钮（对标豆包）
- 语音输入按钮（对标豆包/元宝）
- "联网搜索"作为一个 toggle 开关（对标千问）
- 底部状态栏显示当前 AI，可一键切换

#### 2. 侧边栏 → 折叠式场景导航

```
☰
├ 💬 对话（默认）
├ 📁 文档
│  ├ 上传文件
│  └ 知识库
├ 🤖 能力中心（技能模板）
├ ⏰ 自动化
└ ⚙️ 设置
```

#### 3. 设置 → 三步完成

普通用户模式：

```
⚙️ 设置
┌── 你的 AI ──────────────────────┐
│  智能助手: [✨ 智能推荐 ◀]       │
│  回复风格: [均衡 ◀]             │
│                                  │
│  点击可查看更多 AI 选项           │
│  ┌──────────────────────────┐    │
│  │ ✨ 智能推荐（默认）       │    │
│  │ 🚀 快速响应              │    │
│  │ 🧠 深度思考              │    │
│  │ 🎨 创意模式              │    │
│  └──────────────────────────┘    │
└──────────────────────────────────┘
```

每个选项背后路由到管理员配置好的具体 provider+model，对用户透明。

#### 4. 文档 → 上传即用

```
📁 文档
┌─ 拖拽上传 ─────────────────────┐
│  📥 把文件拖到这里              │
│  或点击选择文件                  │
└────────────────────────────────┘

我的文档：
📄 2026年Q2财报分析.pdf    ✅ AI 已阅读
📄 产品需求文档.docx       ✅ AI 已阅读
📄 团队周报模板.xlsx       🔄 正在阅读...
```

没有"索引"按钮，没有"分块"概念，没有"向量"字样。

#### 5. 自动化 → 场景模板优先

```
⏰ 自动化
┌── 推荐自动化场景 ──────────────┐
│  📅 每日早报    每天早上9点生成  │
│  📊 周报汇总    每周五下午5点    │
│  🔔 新邮件通知  收到新邮件时     │
│  📈 数据看板    每小时更新       │
└──────────────────────────────┘

[创建自动化]  [我的自动化]
```

### Phase 2 — 中间层体验优化（让产品用起来舒服）

#### 1. 语音输入/输出

- 输入框旁加麦克风按钮 → STT 识别 → 发送
- AI 回复可选 TTS 朗读（对标豆包的"朗读"按钮）

#### 2. 连续对话 + 自动记忆

- 用户不需要知道"session"是什么
- 每次打开自动恢复上次对话
- AI 自动记住用户偏好（"我上次说喜欢简洁回答"）

#### 3. 分享功能

- 对话生成分享链接/图片
- 类似豆包的"分享对话"功能

#### 4. 手机/平板适配

- 响应式布局，手机端也能正常使用
- PWA 支持（可添加到桌面）

### Phase 3 — 差异化亮点（竞品做不到的）

#### 1. 上传文档 → 自动关联知识库

用户上传一个合同，AI 自动向量化、提取关键词、关联到 Wiki。用户只看到"合同已存入知识库"，看不到任何技术步骤。

#### 2. "交给 AI 处理" 自动化

用户说"每天帮我整理邮箱里关于项目进度的邮件并生成摘要"→ 系统自动创建工作流，用户全程零配置。所谓"用自然语言创建自动化"。

#### 3. 多 AI 协同但用户无感知

用户在聊天页面只看到输入框和回复。背后可能有 3 个子代理（检索+分析+润色）在并行工作，但对用户来说只是一次提问→一次回答。

## 四、当前版本可立即动手的 Quick Wins

不需要大重构，一两周内能见效的：

| 改动 | 工作量 | 效果 |
|------|--------|------|
| 首页去掉侧边栏打字感，加推荐场景按钮 | 2-3天 | 最大 |
| 将侧边栏 12 项合并为 5 项，改中文场景名 | 1天 | 中 |
| Settings 加"简化模式"：只显示 AI 选择+风格 | 2天 | 大 |
| 知识库去掉"索引"按钮，上传即自动索引 | 1天 | 中 |
| 聊天输入框加语音按钮（前端调用 STT） | 2-3天 | 中 |
| 加首次使用引导浮层（"点击这里开始聊天"） | 1天 | 大 |
