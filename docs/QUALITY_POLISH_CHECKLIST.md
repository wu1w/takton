# Takton 质量打磨清单 v1.0

> 不新功能。只修体验、排隐患、打通断点。

---

## 一、全局缺陷

### 1.1 错误状态不可见

**现状**：操作失败时，要么无提示，要么 toast 一闪而过。

**排查清单**：

| 操作 | 期望 | 现状 |
|------|------|------|
| LLM 调用超时 | "AI 响应超时，正在重试" + 进度条 | ❓ 是卡死还是有提示？ |
| LLM 返回错误（401 密钥失效） | 聊天区的红色提示 + "点击这里重新配置" | ❓ |
| Qdrant 不可达 | 知识库页顶部黄条警告，不阻塞其他功能 | ❓ |
| Qdrant 维度不匹配 | 知识库页红条 + 一键重建方案 | ✅ 小白已做 |
| Cron 执行失败 | 任务状态变红 + 悬停显示错误原因 | ❓ |
| WebSocket 断连 | 顶部黄条"正在重连..." + 加载中状态不阻塞 | ❓ |
| 工作流执行失败 | 画布上节点变红 + 悬停显示错误原因 + 日志面板 | ❓ |
| 知识库索引失败 | 文档卡片显示具体失败原因 + 重试按钮 | ✅ 小白已做 |

### 1.2 加载态缺失

**现状**：大量操作没有加载态，用户不知道"是卡了还是在处理"

**排查清单**：

| 操作 | 期望 | 现状 |
|------|------|------|
| 发送消息后 | 发送按钮变灰/圆形 spinner，"AI 思考中" | ✅ 有 streaming |
| 新建会话 | 侧边栏立即出现占位符 + spinner | ❓ |
| 切换到另一个会话 | 消息区显示骨架屏，同时后台加载 | ❓ |
| Cron 立即执行 | 按钮变"执行中..." + 结果异步推送 | ❓ |
| 工作流运行 | 画布上节点逐个高亮 + 底部日志滚动 | ❓ |
| 拉取模型列表 | 按钮变"拉取中..." + 获取后直接填充下拉 | ✅ 有状态 |
| 文档上传 | 上传进度条 + 文件卡片占位 | ✅ 有迁移拖拽 |

### 1.3 空状态引导

**现状**：很多页面首次打开是空的，但无引导。

| 页面 | 空状态 | 期望 |
|------|--------|------|
| 知识库 - 无文档 | 空白区域 | "还没有文档，点击或拖拽上传" |
| 工作流 - 无工作流 | 空白画布 | "还没有工作流，从[模板]或[对话创建]开始" |
| 定时任务 - 无任务 | 空列表 | "还没有定时任务，创建一个让 AI 自动工作" |
| 会话列表 - 无会话 | 空侧边栏 | "点击 + 开始新对话" |
| 检索测试 - 无结果 | "未找到相关内容" | "该查询未匹配到知识，试试换个问法？或者[上传相关文档]" |
| MCP - 无服务器 | 空列表 | "还没有 MCP 服务器，[添加一个]" |

## 二、功能间流转断点

### 2.1 需要"看完 A 页面去 B 页面操作"的场景

| 场景 | 当前 | 期望 |
|------|------|------|
| 聊天中想用工作流 | 打开新标签页 → 手动记工作流名 → 回聊天 → 输入框选工作流 | **一键跳转**：从聊天输入框的"选择工作流"面板可直达工作流列表去创建 |
| Cron 想绑定工作流 | 去 Cron 页 → 下拉选 → 没有？→ 跳转工作流页创建 | Cron 的下拉框旁边加 [+ 新建工作流] 按钮 |
| Skills 页看到想用的 | 记下名字 → 去 Agent 配置页勾选 | Skills 卡片加 "启用" 按钮，一键加入 agent 配置 |
| 聊天中想看知识库 | 打开新标签 → 知识库 → 搜索 | 聊天输入框 @ 可以搜索知识库文档，输入框内直接搜 |

### 2.2 操作后缺少后续引导

| 操作 | 完成后 | 期望 |
|------|--------|------|
| 创建 Cron | "创建成功" toast → 退出 | 弹出"是否立即执行一次？" |
| 工作流保存 | "已保存" toast → 留在画布 | 弹出"绑定 Cron 定时执行？绑定会话推送结果？" |
| 文档索引完成 | toast 告知分块数 | 弹出"去检索测试验证？" |
| 创建子代理 | "创建成功" | 弹出"是否添加到集群模式？" |
| 修改 Settings | "已保存" | 弹出"用新配置测试一条消息？" |

### 2.3 数据变更后其他页面不感知

| 场景 | 问题 | 修复 |
|------|------|------|
| Settings 切换模型 | 聊天页还在用旧模型发消息 | 聊天页监听 settings 变更事件，自动切换 |
| 知识库增加文档 | RAG 检索立即能查到新内容 | ✅ 没问题 |
| 工作流已删除 | Cron 页面仍显示绑定的工作流 | Cron 加载时校验 workflow_id 是否有效，失效则标记异常 |
| 子代理已禁用 | 集群模式开关仍显示 | 集群模式加载时检查 enabled 状态 |

## 三、边缘场景

### 3.1 快速连续操作

| 场景 | 风险 | 期望 |
|------|------|------|
| 快速连续发送两条消息 | Agent Loop 竞态执行？ | 第二条消息加入队列（已有 loop lock），但输入框无反馈 → 应显示"消息已排队" |
| 快速连续点击"新会话" | 创建重复会话？ | 按钮点击后立即 disabled，创建完成再恢复 |
| 快速连续点击"立即执行"Cron | 重复执行？ | 按钮立即 disabled + "执行中..." |
| 工作流运行中修改 DAG | 执行读取旧版本？新版本混入？ | 执行前 snapshot DAG，运行中编辑标记为"草稿"而非覆盖 |
| 知识库索引中删除文档 | 孤儿 chunks？ | 删除时标记"待清理"并取消后台任务，索引完成后再异步清理 chunk 和向量 |

### 3.2 数据异常情况

| 场景 | 期望 |
|------|------|
| Qdrant 中有向量但对应文档已删除 | 检索结果中过滤掉孤儿 chunk（匹配 document_id 是否存在） |
| Session 的被引用的 workflow 已被删除 | 会话加载时标记"工作流不可用"，不报错 |
| Cron 定时到了但 backward 绑定的模型/供应商已不可用 | 执行前做一次快速连通性检查，失败则记录"模型不可用"并暂停任务 |
| LLM 返回的 DAG 格式非法 | generate_workflow 工具内部做 schema 校验，不合格的返回"生成失败，请重新描述" |

### 3.3 大尺寸数据

| 场景 | 期望 |
|------|------|
| 单文档 > 10MB | 上传前校验，超过限制友好提示 |
| 知识库文档 > 1000 个 chunks | 分页查询 chunks，前端加虚拟滚动 |
| 聊天消息 > 200 条 | 前端加虚拟滚动 / 自动加载更多 |
| Cron 历史 > 100 条 | 分页 + 自动清理旧记录 |

## 四、协同冲突

### 4.1 多标签页 / 多设备同时操作

| 场景 | 风险 | 期望 |
|------|------|------|
| 两个标签页同时发送消息到同一会话 | 两个 Agent Loop 同时执行 | 后端已有 session lock，前端应提示"该会话正在处理中" |
| 两个标签页同时编辑同一工作流 | 后保存的覆盖先保存的 | 乐观锁：保存时传 `updated_at`，后端比较时间戳拒绝过期修改 |
| 两个设备同时修改 Settings | 设置不一致 | 后保存的生效，前保存的被覆盖后 WebSocket 通知刷新 |
| A 删除了子代理，B 正开着集群模式使用它 | B 的无提示的错误 | WebSocket 推送"子代理 XXX 已被删除，已自动移除" |

### 4.2 权限冲突

| 场景 | 期望 |
|------|------|
| 非 owner 用户尝试删除某工作流/Cron | 后端 403，前端按钮置灰不显示 |
| 普通用户尝试修改系统级 Settings | 隐藏高级模式 |

## 五、审计闭环

### 5.1 用户需要知道自己做了什么

| 场景 | 期望 |
|------|------|
| "这段 AI 回复是怎么来的？" | 悬停消息或展开看到：参考了哪些工具调用、是否用了知识库、是否用了工作流 |
| "这个定时任务上次为什么失败了？" | Cron 详情页显示最后 10 次执行记录（时间 + 结果 + 错误原因） |
| "工作流更新了几次？" | 工作流页加"历史版本"（先做简单：记录最近 3 次 DAG 的快照） |

### 5.2 Token 用量可见

| 指标 | 期望位置 |
|------|---------|
| 当前会话用了多少 token | 聊天区底部状态栏 |
| 今天总共用了多少 | 顶部系统状态栏或个人页 |
| 哪些操作消耗最多 | 按"会话/工作流/Cron"分组统计 |

## 六、排查优先级

### P0（必须修 — 会导致功能不可用）

1. LLM 调用失败的错误提示 → 聊天区要有可见的红色提示
2. WebSocket 断连的用户提示
3. 工作流执行失败的可见告警
4. Cron 执行失败的错误原因可见
5. 快速连续操作导致的竞态（发送/新建/执行按钮防抖）

### P1（影响体验流畅度）

6. 全站加载态缺失（Skeleton/Spinner）
7. 全站空状态引导
8. 数据变更后其他页面不感知（刷新/WebSocket 同步）
9. 操作后缺少后续引导（"接下来做什么？"）

### P2（打磨细节）

10. 多标签页/多设备冲突处理
11. 大尺寸数据的性能和限制
12. Token 用量统计
13. 审计日志（消息来源追溯）
14. 边缘数据清理（孤儿 chunks、失效引用）

## 七、建议测试方式

不用写大型测试套件。按场景走一遍：

```
1. 打开两个标签页
2. 同时发送消息 → 期望看到"处理中"提示
3. 一边运行 Cron 一边删除绑定的工作流 → 期望 Cron 变成异常状态而非崩溃
4. 上传大文件 → 期望有限制提示
5. 快速点击 5 次"新会话" → 期望只创建一个
6. 断网 5 秒 → 期望看到重连提示
7. 在设置改模型 → 切回聊天发消息 → 期望用新模型
8. 删除一个在工作流中使用的子代理 → 期望工作流不崩溃
9. Qdrant 容器关掉 → 进知识库 → 期望黄条警告
10. LLM 返回格式错误 → 期望有提示而不是空白回复
```
